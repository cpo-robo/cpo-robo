API KEY SECURITY REMINDER
==========================
Always keep your OpenAI API keys private and secure.

BEST PRACTICES:
1. Go to: https://platform.openai.com/account/api-keys
2. Create a NEW API key for this bot
3. Use the new key with SET_API_KEY.bat
4. Never share keys in chats or commit them to GitHub

================================================================================

HOW TO STORE API KEYS SAFELY
=============================

GOOD (What you should do):
✓ Use SET_API_KEY.bat - stores in Windows environment variables
✓ Environment variables are secure and per-computer
✓ Won't be exposed in code files
✓ Settings persist across sessions

BAD (Don't do this):
✗ Never share API keys in chats
✗ Never commit keys to git/GitHub
✗ Never hardcode keys in Python files
✗ Never paste keys in unsecured locations

================================================================================

USING SET_API_KEY.BAT (SAFE METHOD)
===================================

1. Double-click SET_API_KEY.bat
2. Paste your NEW OpenAI API key when prompted
3. Your key is stored in Windows environment variables
4. It's never visible in code
5. It persists automatically

The bot will use this key securely.

================================================================================

IF YOU USED THE EXPOSED KEY ALREADY
===================================

Don't worry! Here's what to do:

1. REVOKE the exposed key immediately:
   - Go to https://platform.openai.com/account/api-keys
   - Delete the key I listed above

2. CREATE a new key:
   - Same page, click "Create new secret key"
   - Copy the new key (it won't show again!)

3. UPDATE your bot:
   - Run SET_API_KEY.bat
   - Paste your NEW key
   - Start using the bot

4. That's it! The old key is useless now.
